
### Use Cases

**1. Register for an Account and Create Profile**
*   **Actor:** Participant, Judge, Mentor
*   **Description:** A user can register for an account in the system. After registration, they can create and update their profile with information such as skills, experience, and contact details.

**2. View Hackathons**
*   **Actor:** All Users
*   **Description:** Any user can view a list of upcoming and ongoing hackathons, including details like the theme, schedule, and prizes.

**3. Join a Hackathon**
*   **Actor:** Participant
*   **Description:** A participant can register to join a specific hackathon.

**4. Create and Manage a Team**
*   **Actor:** Participant
*   **Description:** A participant can create a new team or join an existing one. The team creator becomes the team leader and can invite other participants to join.

**5. Submit Project**
*   **Actor:** Participant (Team)
*   **Description:** A team can submit their project for a hackathon. The submission includes a project description, source code repository link, and a video demo.

**6. Request Mentorship**
*   **Actor:** Participant (Team)
*   **Description:** A team can request help and guidance from a mentor.

**7. View and Score Submissions**
*   **Actor:** Judge, Guest_Judge
*   **Description:** A judge or guest judge can view the list of projects submitted for a hackathon and score them based on predefined criteria.

**8. Provide Feedback**
*   **Actor:** Judge, Guest_Judge, Mentor
*   **Description:** A judge, guest judge, or mentor can provide feedback on a submitted project.

**9. Manage Hackathon**
*   **Actor:** Organizer
*   **Description:** An organizer can create and manage a hackathon, including setting up the schedule, defining the rules, and managing the list of participants.

**10. Manage Users**
*   **Actor:** Organizer, Admin
*   **Description:** An organizer can manage the users participating in their hackathon. An admin can manage all users in the system.

**11. Manage Roles and Permissions**
*   **Actor:** Admin
*   **Description:** An admin can manage the roles and permissions of all users in the system.

**12. View Reports**
*   **Actor:** Organizer
*   **Description:** An organizer can view reports and analytics about their hackathon, such as the number of participants, the number of submissions, and the average scores.
