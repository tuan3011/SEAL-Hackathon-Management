
### Functional Requirements

**1. User Management**
    1.1. Users shall be able to register for an account using their email address.
    1.2. The system shall support the following user roles: Participant, Judge, Guest_Judge, Mentor, Organizer, Admin.
    1.3. Users shall be able to log in and log out of the system.
    1.4. Users shall be able to view and edit their own profiles.
    1.5. Admins shall be able to manage all user accounts.

**2. Hackathon Management**
    2.1. Organizers shall be able to create, update, and delete hackathons.
    2.2. Each hackathon shall have a name, description, start date, and end date.
    2.3. The system shall display a list of all upcoming, ongoing, and past hackathons.

**3. Team Management**
    3.1. Participants shall be able to create and join teams.
    3.2. Each team shall have a name and a list of members.
    3.3. Team leaders shall be able to manage their team members.

**4. Submission Management**
    4.1. Teams shall be able to submit their projects for a hackathon.
    4.2. Each submission shall include a project description, a link to a code repository, and an optional video demo.
    4.3. Submissions shall only be accepted within the hackathon's submission period.

**5. Judging and Scoring**
    5.1. Judges shall be able to view all submissions for a hackathon.
    5.2. Judges shall be able to score submissions based on predefined criteria.
    5.3. The system shall calculate the final score for each submission.

**6. Mentorship**
    6.1. Teams shall be able to request mentorship.
    6.2. Mentors shall be able to view and accept mentorship requests.

**7. Reporting**
    7.1. Organizers shall be able to view reports on hackathon statistics, such as the number of participants and submissions.
